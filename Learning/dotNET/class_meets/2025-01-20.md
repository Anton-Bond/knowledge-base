Основные принципы REST API (бэст практис):
  - ресурсы,
  - уник индентификаторы,
  - стандартизация, 
  - чистые url, локаничные url
  - использовать статусы, правильные коды
 - документация: swagger и другие
  - безопасность (аутенфикация)
  - реализация обработки ошибок
  - кэширование (особ высконаграженные)
  - иденпотентность (не будет сайдэфектов)
  - неид методы - (пост, чаще всего пэдж)
  - и дригие
