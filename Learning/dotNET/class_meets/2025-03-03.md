!!! SOLID принципы
S -каждый класс имеет только одну причину для изменений
(формулировка ошбочная: каждый класс выполняет одну задачу)
O - код открыт для расширения но закрыт для модификации
L - принцип подстановки БЛ - дочерний класс должен заменять родительский без изменений логики
I - интерфейсы должны быть узкими и не громоздкими (прицип српи для интерфейсов)
D - зависимоти должны быть абстракции а не реализации
