### **1. Непотокобезопасные коллекции:**

Эти коллекции не гарантируют корректной работы при доступе из нескольких потоков одновременно. Использовать их нужно с внешней синхронизацией.

**Примеры:**

- `List<T>`
- `Dictionary<TKey, TValue>`
- `Queue<T>`
- `Stack<T>`

**Риски:**

- Ошибки гонок (race conditions).
- Потеря данных или некорректные результаты.

**Решение:**  
Используйте внешнюю блокировку:

```csharp
lock(lockObject)
{
    myList.Add(item);
}
```

---

### **2. Потокобезопасные коллекции:**

Гарантируют корректную работу при доступе из нескольких потоков без необходимости дополнительной синхронизации.

**Примеры:**

- `ConcurrentDictionary<TKey, TValue>`
- `ConcurrentQueue<T>`
- `ConcurrentStack<T>`
- `BlockingCollection<T>`

**Преимущества:**

- Встроенная поддержка многопоточности.
- Высокая производительность, так как используют минимальную блокировку или неблокирующие алгоритмы.

**Пример использования:**

```csharp
var concurrentDict = new ConcurrentDictionary<int, string>();
concurrentDict.TryAdd(1, "Value1");
Console.WriteLine(concurrentDict[1]); // "Value1"
```

### **Выбор коллекции:**

- Используйте **потокобезопасные коллекции** в многопоточной среде.
- Используйте **непотокобезопасные коллекции** для однопоточных операций, чтобы избежать избыточных накладных расходов.
