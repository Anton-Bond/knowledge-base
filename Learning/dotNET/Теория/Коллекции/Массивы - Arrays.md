**Коллекция фиксированного размера, содержащая элементы одного типа.**

**Особенности:**
- Быстрый доступ к элементам по индексу.
- Нельзя изменять размер после создания.
- безопасная коллекция
- не потокобезопасный

```C#
int[] nums2 = new int[4] { 1, 2, 3, 5 };
int[] nums3 = new int[] { 1, 2, 3, 5 };
int[] nums4 = new[] { 1, 2, 3, 5 };
int[] nums5 = { 1, 2, 3, 5 };
int[] nums1 = [ 1, 2, 3, 5 ];
int[] nums2 = [];   // пустой массив
```

### Многомерные массивы

Массивы характеризуются таким понятием как ранг или количество измерений. Выше мы рассматривали массивы, которые имеют одно измерение (то есть их ранг равен 1) - такие массивы можно представлять в виде ряда (строки или столбца) элемента. Но массивы также бывают многомерными.
```C#
int[] nums1 = new int[] { 0, 1, 2, 3, 4, 5 };
int[,] nums2 = { { 0, 1, 2 }, { 3, 4, 5 } };
int[,] nums1;
int[,] nums2 = new int[2, 3];
int[,] nums3 = new int[2, 3] { { 0, 1, 2 }, { 3, 4, 5 } };
int[,] nums4 = new int[,] { { 0, 1, 2 }, { 3, 4, 5 } };
int[,] nums5 = new [,]{ { 0, 1, 2 }, { 3, 4, 5 } };
int[,] nums6 = { { 0, 1, 2 }, { 3, 4, 5 } };
int[,,] nums3 = new int[2, 3, 4];
```
**длина такого массива - это совокупное количество элементов**

### Массив массивов

От многомерных массивов надо отличать массив массивов или так называемый "зубчатый массив":
```C#
int[][] nums = new int[3][];
nums[0] = new int[2] { 1, 2 };          // выделяем память для первого подмассива
nums[1] = new int[3] { 1, 2, 3 };       // выделяем память для второго подмассива
nums[2] = new int[5] { 1, 2, 3, 4, 5 }; // выделяем память для третьего подмассива

int[][] numbers = { 
    new int[] { 1, 2 }, 
    new int[] { 1, 2, 3 }, 
    new int[] { 1, 2, 3, 4, 5 } 
};
```

### Основные понятия массивов

Суммируем основные понятия массивов:
- Ранг (rank): количество измерений массива
- Длина измерения (dimension length): длина отдельного измерения массива
- Длина массива (array length): количество всех элементов массива


1. **Вопрос:** Чем массив отличается от `List`?  
    **Ответ:**
    - Массив имеет фиксированный размер, а `List` — динамический.
    - Массив не поддерживает встроенные методы для добавления/удаления элементов.
    - Производительность массива выше, так как он не содержит дополнительных методов управления.
2. **Вопрос:** Как создать двумерный массив?  
    **Ответ:**    
    `int[,] matrix = new int[2, 3] { { 1, 2, 3 }, { 4, 5, 6 } }; Console.WriteLine(matrix[1, 2]); // Вывод: 6`
    
3. **Вопрос:** Что такое зубчатый массив (jagged array)?  
    **Ответ:**  
    Это массив массивов, где подмассивы могут иметь разную длину:
     `int[][] jagged = new int[2][];`
     `jagged[0] = new int[] { 1, 2 };` 
     `jagged[1] = new int[] { 3, 4, 5 };`
    
4. **Вопрос:** Можно ли изменить размер массива?  
    **Ответ:** Нет, размер массива фиксирован. Но можно использовать `Array.Resize` для создания нового массива:
    `int[] arr = { 1, 2, 3 }; Array.Resize(ref arr, 5);`
    
5. **Вопрос:** Как скопировать массив?  
    **Ответ:**
    - Поверхностное копирование: `Array.Copy`.
    - Глубокое копирование требует клонирования объектов, если массив содержит ссылки.
6. **Вопрос:** Что такое `ArraySegment`?  
    **Ответ:** Это структура, позволяющая работать с подмножеством массива без его копирования.
    
7. **Вопрос:** Как найти максимальное/минимальное значение в массиве?  
    **Ответ:** Используйте LINQ:
    `int max = arr.Max();`
    `int min = arr.Min();`
    
8. **Вопрос:** Как перевернуть массив?  
    **Ответ:**
    `Array.Reverse(arr);`
    
9. **Вопрос:** Как работает метод `Array.Sort`?  
    **Ответ:** Он сортирует массив по возрастанию с использованием алгоритма QuickSort.
    
10. **Вопрос:** Можно ли массив сделать потокобезопасным?  
    **Ответ:** Сам массив не потокобезопасен, но можно синхронизировать доступ к нему с помощью блокировок.
