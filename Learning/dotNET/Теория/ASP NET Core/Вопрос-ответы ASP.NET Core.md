#### 1. **Что такое ASP.NET Core?**

**Ответ:**  
ASP.NET Core — это кроссплатформенный, высокопроизводительный фреймворк с открытым исходным кодом для разработки современных веб-приложений, API и облачных сервисов. Основные особенности: модульность, DI, middleware и поддержка Razor Pages.

---

#### 2. **Какие преимущества ASP.NET Core перед ASP.NET?**

**Ответ:**

- Кроссплатформенность (Windows, Linux, macOS).
- Высокая производительность.
- Модульная архитектура.
- Поддержка контейнеров, таких как Docker.
- Интеграция с современными инструментами, такими как Blazor.
-  asp.net core имеет более гибкую настройку, в asp.net только в xml file
- asp.net core - open source (это определяет развитие платформы и большое комьюнити)

---

#### Что такое Blazor?

**Blazor** — это фреймворк от Microsoft, входящий в экосистему ASP.NET Core, который позволяет разрабатывать современные веб-приложения с использованием C# вместо JavaScript. Blazor использует библиотеку Razor для работы с компонентами, что делает разработку похожей на работу с HTML и Razor Pages.

---

#### 3. **Что такое Middleware?**

**Ответ:**  
Middleware — это компонент, который обрабатывает HTTP-запрос или модифицирует его, передавая дальше по конвейеру. Примеры встроенных middleware: `Authentication`, `Authorization`, `Routing`.

---

#### 4. **Как работает конвейер (pipeline) в ASP.NET Core?**

**Ответ:**  
Запрос проходит через серию middleware в порядке их регистрации. Каждый middleware может:

1. Выполнить логику.
2. Передать управление следующему middleware.
3. Вернуть ответ клиенту.

---

#### 5. **Как зарегистрировать middleware в ASP.NET Core?**

**Ответ:**  
Middleware регистрируются в методе `Configure` через вызовы `app.Use...`:

```csharp
app.UseRouting();
app.UseAuthentication();
app.UseAuthorization();
```

---
#### **Шаги для создания кастомного middleware**

1. **Создайте класс для middleware.**
2. **Реализуйте метод `Invoke` или `InvokeAsync`.**
3. **Зарегистрируйте middleware в конвейере в методе `Configure` приложения.**
```C#
app.UseMiddleware<RequestTimingMiddleware>();
```

---

#### 6. **Что такое встроенные middleware?**

**Ответ:**  
Встроенные middleware — это стандартные компоненты, такие как:

- `UseRouting`: управление маршрутизацией.
- `UseStaticFiles`: обслуживание статических файлов.
- `UseAuthentication`: аутентификация.
- `UseAuthorization`: авторизация.

---

#### 7. **Как настроить маршруты в ASP.NET Core?**

**Ответ:**  
Маршруты настраиваются с помощью метода `UseEndpoints`:

```csharp
app.UseEndpoints(endpoints =>
{
    endpoints.MapControllerRoute(
        name: "default",
        pattern: "{controller=Home}/{action=Index}/{id?}");
});
```

---

#### 8. **Чем отличаются шаблонные маршруты и атрибутные маршруты?**

**Ответ:**

- Шаблонные маршруты определяются глобально в `UseEndpoints`.
- Атрибутные маршруты задаются в контроллерах с помощью атрибутов `[Route]`, `[HttpGet]`, `[HttpPost]` и др.

---

#### 9. **Что такое MapControllerRoute()?**

**Ответ:**  
`MapControllerRoute` регистрирует маршруты для контроллеров, используя заданный шаблон URL.

---

#### 10. **Какие ограничения можно задать для маршрутов?**

**Ответ:**  
Ограничения задают допустимые значения параметров:

- `{id:int}` — только целые числа.
- `{name:alpha}` — только буквы.
- `{value:minlength(3)}` — минимальная длина.

---

#### 11. **Что такое Razor Pages?**

**Ответ:**  
Razor Pages — это упрощенный способ организации веб-приложений, где каждая страница обрабатывает свой маршрут и содержит как логику, так и представление.

---

#### 12. **Чем Razor Pages отличаются от MVC?**

**Ответ:**

- Razor Pages работают на уровне страниц, а не контроллеров.
- Логика находится в файле `PageModel`, связанном с `.cshtml`.
- Подходит для проектов, ориентированных на страницы, а не API.

---

#### 13. **Как передать параметры в Razor Pages?**

**Ответ:**  
 - Параметры маршрута (Route Parameters)
 - Параметры строки запроса (Query Parameters)
 - Использование `TempData` (Это полезно при редиректе с одной страницы на другую.)
 - Использование формы или привязки модели

---

#### 14. **Что такое PageModel в Razor Pages?**

**Ответ:**  
`PageModel` — это класс, который содержит логику для Razor Page. Он наследуется от `PageModel` и обрабатывает запросы через методы `OnGet`, `OnPost` и др.

---

#### 15. **Как обрабатывать формы в Razor Pages?**

**Ответ:**  
Используя привязку модели с атрибутом `[BindProperty]` и метод `OnPost`:

```csharp
[BindProperty]
public string Name { get; set; }

public void OnPost()
{
    // Обработка данных
}
```

---

#### 16. **Что такое контроллер в ASP.NET Core?**

**Ответ:**  
Контроллер — это класс, который обрабатывает входящие запросы, выполняет логику и возвращает ответ (HTML, JSON, редирект и т.д.).

---

#### 17. **Какой базовый класс используется для контроллеров?**

**Ответ:**  
Контроллеры наследуются от `Controller` (для MVC) или `ControllerBase` (для API).

---

#### 18. **Какие типы возвращаемых данных поддерживают действия контроллеров?**

**Ответ:**

- `ViewResult` — возвращает HTML-страницу.
- `JsonResult` — JSON-данные.
- `ContentResult` — текст.
- `RedirectResult` — редирект.
- `OkResult`, `NotFoundResult` и другие — для API.

---

#### 19. **Как работает атрибут `[Route]`?**

**Ответ:**  
`[Route]` задает маршрут для контроллера или метода:

```csharp
[Route("api/products")]
public IActionResult GetProducts() { }
```

---

#### 20. **Чем отличается `[HttpGet]` от `[Route]`?**

**Ответ:**

- `[HttpGet]` указывает метод HTTP, для которого действие доступно.
- `[Route]` задает URL.

---

#### 21. **Что такое фильтры?**

**Ответ:**  
Фильтры — это механизмы, выполняющие логику до или после метода действия. Примеры: `AuthorizationFilter`, `ActionFilter`, `ExceptionFilter`.

---

#### 22. **Как использовать Dependency Injection (DI) в контроллерах?**

**Ответ:**  
DI внедряется через конструктор:

```csharp
public class HomeController : Controller
{
    private readonly ILogger<HomeController> _logger;

    public HomeController(ILogger<HomeController> logger)
    {
        _logger = logger;
    }
}
```

---

#### 23. **Что такое жизненный цикл контроллера?**

**Ответ:**

1. Получение запроса.
2. Создание экземпляра контроллера.
3. Вызов фильтров.
4. Выполнение метода действия.
5. Формирование ответа.

#### 24. **Как настроить авторизацию в ASP.NET Core?**

**Ответ:**  
Используя middleware:

```csharp
app.UseAuthentication();
app.UseAuthorization();
```

---

#### 25. **Какие провайдеры аутентификации поддерживаются?**

**Ответ:**  
OAuth, OpenID Connect, JWT, Facebook, Google, Microsoft.

---

#### 26. **Как регистрируются сервисы в ASP.NET Core?**

**Ответ:**  
Сервисы регистрируются в `Program.cs`:

```csharp
builder.Services.AddScoped<IMyService, MyService>();
```

---

#### 27. **Как включить поддержку статических файлов?**

**Ответ:**  
Использовать middleware `UseStaticFiles`:

```csharp
app.UseStaticFiles();
```

---

#### 28. **Что такое модель привязки (Model Binding)?**

**Ответ:**  
Процесс автоматического преобразования данных из запросов (формы, строки запроса, тела) в параметры метода или свойства модели.

---

#### 29. **Что такое Tag Helpers?**

**Ответ:**  
Tag Helpers позволяют использовать C#-логику в HTML с помощью специальных атрибутов. Пример:

```html
<form asp-action="Submit">
```

---

#### 30. **Как настроить логирование?**

**Ответ:**  
Логирование настраивается в `appsettings.json` или коде:

```csharp
builder.Logging.AddConsole();
```
