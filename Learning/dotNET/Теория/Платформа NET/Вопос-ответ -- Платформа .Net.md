### **Вопросы и ответы для интервью на позицию Middle-разработчика по платформе .NET**

---

#### **Базовые вопросы**

1. **Что такое .NET? В чем его основное назначение?**  
    .NET — это универсальная платформа для разработки, предоставляющая среду выполнения, библиотеки и инструменты для создания приложений на различных языках, таких как C#, F#, и VB.NET. Основное назначение — упрощение разработки и обеспечения кроссплатформенности.
    
2. **Какие основные версии .NET вы знаете?**
    
    - .NET Framework
    - .NET Core
    - .NET 5 и выше
3. **Чем отличается .NET Framework от .NET Core?**  
    .NET Framework предназначен для Windows, тогда как .NET Core — кроссплатформенный и более производительный. .NET Core также модульный и подходит для облачных решений.
    
4. **Что такое современный .NET (5 и выше)?**  
    Это унифицированная платформа, объединяющая возможности .NET Core и .NET Framework, с улучшенной производительностью и кроссплатформенностью.
    
5. **Что такое CLR и для чего он используется?**  
    Common Language Runtime (CLR) — это среда выполнения, которая управляет выполнением приложений .NET, обеспечивает сборку мусора, управление памятью и безопасность кода.
    

---

#### **Вопросы о функциях и технологиях .NET**

6. **Что такое сборка (assembly) в .NET?**  
    Сборка — это основная единица развертывания и версии в .NET, которая содержит метаданные, исполняемый код и ресурсы.
    
7. **Какую роль выполняют библиотеки классов в .NET?**  
    Библиотеки классов (Base Class Library, BCL) предоставляют стандартный набор API для работы с файлами, строками, сетями и другими функциями.
    
8. **Что такое пакет NuGet?**  
    NuGet — это менеджер пакетов для .NET, который позволяет добавлять, обновлять и управлять сторонними библиотеками и зависимостями.
    
9. **Что такое LINQ? Как он упрощает работу с данными?**  
    LINQ (Language Integrated Query) предоставляет унифицированный синтаксис для работы с данными (коллекции, базы данных, XML и т. д.) прямо в коде C#.
    
10. **Какие виды коллекций вы знаете в .NET?**
    -- Коллекции на основе массивов (Array, List).
    -- Связные коллекции (LinkedList).
    -- Словари (Dictionary<TKey, TValue>).
    -- Очереди и стеки (Queue, Stack).

---

#### **Кроссплатформенность и совместимость**

11. **Как обеспечить кроссплатформенность в .NET Core или .NET 5+?**  
    Кроссплатформенность обеспечивается за счёт унифицированной среды выполнения (runtime) и возможности компиляции для Windows, Linux и macOS.
    
12. **Что такое .NET Standard?**  
    .NET Standard — это спецификация, обеспечивающая совместимость библиотек между разными версиями .NET (Framework, Core, 5+).
    
13. **Как проверить совместимость проекта перед миграцией на .NET 5 или 6?**  
    Использовать [Portability Analyzer](https://aka.ms/dotnet-portability) или .NET Upgrade Assistant для анализа API и зависимостей.
    
14. **Какие проблемы могут возникнуть при миграции с .NET Framework на .NET Core или .NET 6?** 

- Отсутствие поддержки некоторых устаревших API.
- Зависимость от библиотек, не поддерживающих новую платформу.
- Необходимость переписывания кода.

15. **Какой инструмент используется для автоматизации миграции проектов?**  
    .NET Upgrade Assistant.

#### **Производительность и оптимизация**

16. **Что такое JIT и AOT? Чем они отличаются?**

- JIT (Just-In-Time) компилирует код во время выполнения.
- AOT (Ahead-Of-Time) компилирует код заранее, что улучшает скорость запуска приложения.

17. **Как реализована сборка мусора (GC) в .NET?**  
    Сборка мусора автоматически управляет памятью, удаляя объекты, которые больше не используются, через поколения (Generation 0, 1, 2).
    
18. **Что такое async/await в .NET?**  
    Это ключевые слова для реализации асинхронного программирования, которые позволяют выполнять задачи без блокировки основного потока.
    
19. **Как измерить производительность приложения в .NET?**
    

- Использовать профайлеры (например, Visual Studio Profiler, dotTrace).
- Встроенные классы: `Stopwatch` и `PerformanceCounter`.

20. **Какие инструменты вы используете для мониторинга приложений в .NET?**

- Application Insights.
- ELK Stack (Elasticsearch, Logstash, Kibana).
- Prometheus и Grafana.

#### **Облачные и веб-технологии**

21. **Что такое ASP.NET Core?**  
    Кроссплатформенный фреймворк для разработки веб-приложений, обладающий высокой производительностью и поддержкой контейнеров.
    
22. **Какие основные особенности Blazor?**  
    Blazor позволяет создавать клиентские веб-приложения с использованием C# вместо JavaScript, поддерживает WebAssembly и серверную модель.
    
23. **Что такое Kestrel?**  
    Лёгкий веб-сервер, используемый ASP.NET Core для обработки HTTP-запросов.
    
24. **Какие форматы сериализации поддерживает .NET?**  
    JSON (через System.Text.Json), XML, Binary, Protocol Buffers.
    
25. **Что такое Dependency Injection?**  
    Dependency Injection (DI) — это паттерн проектирования, который позволяет управлять зависимостями между объектами через внедрение.
    

---

#### **Безопасность и тестирование**

26. **Как реализовать безопасность в ASP.NET Core?**

- Использовать аутентификацию и авторизацию (JWT, OAuth, OpenID Connect).
- Защита данных с помощью ASP.NET Data Protection.
- Использовать HTTPS и политики CORS.

27. **Какие инструменты вы используете для тестирования .NET-приложений?**

- xUnit, NUnit, MSTest для модульного тестирования.
- Moq или NSubstitute для мокирования.

28. **Как обрабатывать ошибки в .NET?**

- Использовать `try-catch-finally`.
- Логирование через ILogger.
- Настраивать Middleware для обработки исключений в ASP.NET Core.

29. **Что такое Middleware в ASP.NET Core?**  
    Компоненты промежуточного уровня, которые обрабатывают запросы и ответы в конвейере.
    
30. **Как защищать конфиденциальные данные в приложении?**
    

- Использовать Secret Manager для хранения конфигураций.
- Шифровать конфиденциальные данные.
- Настраивать доступ на основе ролей.
