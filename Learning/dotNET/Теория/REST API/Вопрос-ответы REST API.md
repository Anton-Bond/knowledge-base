#### 1. **Что такое REST API?**

**Ответ:**  
REST API — это интерфейс, который позволяет клиентам взаимодействовать с сервером через HTTP, опираясь на принципы REST: ресурсы, методы HTTP, кэширование, состояние без сохранения и унифицированные URL.

---

#### 2. **Чем отличается REST API от RESTful API?**

**Ответ:**  
REST API — это общий термин для API, которые используют REST-принципы. RESTful API строго следует всем принципам REST, включая правильное использование методов HTTP, чистые URL, кэширование и т.д.

---

#### 3. **Какие основные методы используются в REST API?**

**Ответ:**

- **GET**: Получение ресурса.
- **POST**: Создание ресурса.
- **PUT**: Полное обновление ресурса.
- **PATCH**: Частичное обновление ресурса.
- **DELETE**: Удаление ресурса.

---

#### 4. **Какая разница между PUT и PATCH?**

**Ответ:**

- **PUT**: Полностью заменяет весь ресурс. Если ресурса нет, он создается.
- **PATCH**: Изменяет только указанные поля ресурса, оставляя остальные неизменными.

---

#### 5. **Что такое идемпотентность? Какие методы HTTP идемпотентны?**

**Ответ:**  
Идемпотентность означает, что повторное выполнение запроса приведет к одному и тому же результату.  
Идемпотентные методы: **GET**, **PUT**, **DELETE**, **OPTIONS**, **HEAD**.  
Неидемпотентные методы: **POST**, **PATCH**.

---

#### 6. **Какие основные HTTP-статусы вы знаете?**

**Ответ:**

- **2xx** (успех):
    - `200 OK` — Запрос выполнен.
    - `201 Created` — Ресурс создан.
    - `204 No Content` — Успех без тела ответа.
- **4xx** (ошибки клиента):
    - `400 Bad Request` — Неверный запрос.
    - `401 Unauthorized` — Требуется авторизация.
    - `404 Not Found` — Ресурс не найден.
- **5xx** (ошибки сервера):
    - `500 Internal Server Error` — Общая ошибка сервера.
    - `503 Service Unavailable` — Сервис временно недоступен.

---

#### 7. **Какие есть принципы REST API?**

**Ответ:**

- Использование ресурсов.
- Уникальные идентификаторы для каждого ресурса (например, `/users/123`).
- Методы HTTP для работы с ресурсами (GET, POST, PUT, DELETE).
- Чистые и понятные URL.
- Кэширование данных.
- Обработка ошибок с использованием HTTP-статусов.
- Документирование API (например, Swagger).
- Безопасность через HTTPS и аутентификацию.

---

#### 8. **Что такое документация API и почему она важна?**

**Ответ:**  
Документация API — это описание того, как работать с API, включая методы, URL, параметры и ответы. Она важна, чтобы разработчики могли быстро интегрировать и использовать API. Инструменты, такие как Swagger и OpenAPI, помогают создавать и поддерживать документацию.

---

#### 9. **Какие уровни безопасности используются в REST API?**

**Ответ:**

- Использование HTTPS для шифрования данных.
- Аутентификация:
    - **Basic Auth** — Передача логина и пароля.
    - **Bearer Token** (например, JWT) — Использование токенов для проверки подлинности.
    - **OAuth2** — Протокол для безопасного доступа.
- Ограничение количества запросов (rate limiting).

---

#### 10. **Как вы обрабатываете ошибки в REST API?**

**Ответ:**

- Возвращаю подходящий HTTP-статус (`400`, `404`, `500` и т.д.).
- В ответе указываю описание ошибки. Например:
    
    ```json
    {
      "error": "Invalid input",
      "status": 400
    }
    ```
    
- Логирую ошибки на сервере для анализа.

---

#### 11. **Как реализовать кэширование в REST API?**

**Ответ:**

- Использую HTTP-заголовки:
    - `Cache-Control` — для указания политики кэширования.
    - `ETag` — для проверки, изменился ли ресурс.
- Внедряю внешние системы кэширования, такие как Redis или Memcached.

---

#### 12. **Почему POST — неидемпотентный метод?**

**Ответ:**  
POST создает новый ресурс на сервере, и повторный запрос приведет к созданию еще одного ресурса. Например, если вы отправите форму дважды, вы получите два созданных объекта.

---

#### 13. **Какие есть best practices для REST API?**

**Ответ:**

- Чистые URL: используйте понятные пути, например, `/users/123`.
- Используйте подходящие методы HTTP для операций.
- Документируйте API.
- Применяйте HTTPS и методы аутентификации.
- Используйте правильные HTTP-статусы для обработки ошибок.
- Реализуйте кэширование для снижения нагрузки.
- Убедитесь в идемпотентности соответствующих методов.

---

#### 14. **Как поддерживать версии в REST API?**

**Ответ:**  
Добавляйте номер версии в URL:

```plaintext
GET /v1/users  
GET /v2/users  
```

Это позволяет обновлять API без нарушения работы старых клиентов.

---

#### 15. **Что такое Swagger?**

**Ответ:**  
Swagger — это инструмент для документирования REST API. Он позволяет автоматически генерировать документацию, тестировать запросы и делиться API со сторонними разработчиками.
