Abstraction

**Сокрытие сложностей реализации и предоставление только необходимых деталей.**

 - Использование абстрактных классов и интерфейсов.

Абстрактные классы:
```C#
abstract class Shape
{
    public abstract double CalculateArea();
}

class Circle : Shape
{
    public double Radius { get; set; }

    public Circle(double radius)
    {
        Radius = radius;
    }

    public override double CalculateArea()
    {
        return Math.PI * Radius * Radius;
    }
}
```

Интерфейсы:
```C#
interface IMovable
{
    void Move();
}

class Car : IMovable
{
    public void Move()
    {
        Console.WriteLine("The car moves forward.");
    }
}
```


### Абстрактные члены классов

Кроме обычных свойств и методов абстрактный класс может иметь абстрактные члены классов, которые определяются с помощью ключевого слова abstract и не имеют никакого функционала. В частности, абстрактными могут быть:
- Методы
- Свойства
- Индексаторы
- События

Абстрактные члены классов не должны иметь модификатор private. При этом производный класс обязан переопределить и реализовать все абстрактные методы и свойства, которые имеются в базовом абстрактном классе. При переопределении в производном классе такой метод или свойство также объявляются с модификатором override (как и при обычном переопределении виртуальных методов и свойств). Также следует учесть, что если класс имеет хотя бы один абстрактный метод (или абстрактные свойство, индексатор, событие), то этот класс должен быть определен как абстрактный.

Абстрактные члены также, как и виртуальные, являются частью полиморфного интерфейса. Но если в случае с виртуальными методами мы говорим, что класс-наследник наследует реализацию, то в случае с абстрактными методами наследуется интерфейс, представленный этими абстрактными методами.

#### Вопросы:

1. Что такое абстракция?  
    **Ответ:** Это процесс сокрытия сложностей реализации и предоставления только необходимых деталей.
    
2. Чем отличается абстрактный класс от интерфейса?  
    **Ответ:** Абстрактный класс может содержать частичную реализацию, а интерфейс — только объявления методов.
    
3. Какие ключевые слова используются для абстракции в C#?  
    **Ответ:** `abstract` для классов и методов, `interface` для интерфейсов.
    
4. Можно ли создать экземпляр абстрактного класса?  
    **Ответ:** Нет, экземпляр абстрактного класса создать нельзя.
    
5. Что произойдёт, если класс реализует интерфейс, но не реализует его методы?  
    **Ответ:** Класс станет абстрактным.
    
6. Можно ли наследовать абстрактный класс?  
    **Ответ:** Да, абстрактные классы предназначены для наследования.
    
7. Можно ли использовать модификаторы доступа в интерфейсах?  
    **Ответ:** В интерфейсах C# 8.0 и выше можно использовать `public`, `private`, `protected`, но только для членов с реализацией.
    
8. Чем отличается абстрактный метод от виртуального?  
    **Ответ:** Абстрактный метод не имеет реализации, а виртуальный — имеет.
    
9. Можно ли создать конструктор в абстрактном классе?  
    **Ответ:** Да, но только для вызова из производных классов.
    
10. Какие преимущества даёт абстракция?  
    **Ответ:** Упрощение разработки, уменьшение зависимости от реализации.
    
11. Что такое "контракт" в программировании?  
    **Ответ:** Это набор методов и свойств, которые должен реализовать класс (например, интерфейс).
    
12. Как абстракция связана с принципом DRY?  
    **Ответ:** Абстракция позволяет определить общий функционал в одном месте и переиспользовать его.
    
13. Можно ли унаследовать несколько абстрактных классов?  
    **Ответ:** Нет, C# поддерживает наследование только одного класса.
    
14. Какие модификаторы доступа поддерживаются в абстрактных классах?  
    **Ответ:** Все стандартные модификаторы доступа: `public`, `private`, `protected`, `internal`.
    
15. Как абстракция влияет на тестирование кода?  
    **Ответ:** Она упрощает создание mock-объектов и модульное тестирование.
    
16. Можно ли добавить поля в интерфейс?  
    **Ответ:** Нет, но в C# 8.0 и выше можно использовать поля в интерфейсах с реализацией по умолчанию.
    
17. Что такое реализация по умолчанию в интерфейсах?  
    **Ответ:** Это методы с реализацией внутри интерфейсов (начиная с C# 8.0).
    
18. Можно ли использовать абстрактный класс без абстрактных методов?  
    **Ответ:** Да, но это не рекомендуется.
    
19. Как абстракция помогает в масштабируемости?  
    **Ответ:** Она изолирует изменения в реализации от внешнего интерфейса.
    
20. Как связаны абстракция и инкапсуляция?  
    **Ответ:** Абстракция скрывает сложность на уровне интерфейса, а инкапсуляция — на уровне реализации.
