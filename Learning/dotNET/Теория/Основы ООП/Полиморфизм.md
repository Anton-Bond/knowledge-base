Polymorphism

**Возможность использовать один интерфейс для работы с разными типами объектов.**

**Типы:**
- **Компиляторный (overloading):** Перегрузка методов с разными сигнатурами.
- **Рантаймовый (overriding):** Переопределение методов базового класса в производных.

```C#
public class Shape
{
    public virtual void Draw()
    {
        Console.WriteLine("Drawing a shape.");
    }
}

public class Circle : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a circle.");
    }
}
```

```C#
Shape shape = new Circle();
shape.Draw(); // Вывод: Drawing a circle.
```

#### Вопросы:

1. Что такое полиморфизм?  
    **Ответ:** Полиморфизм — это способность объектов обрабатывать вызовы одного и того же метода по-разному в зависимости от их типа.
    
2. Какие виды полиморфизма есть в C#?  
    **Ответ:** Компиляторный (перегрузка) и рантаймовый (переопределение).
    
3. Как реализуется полиморфизм времени выполнения?  
    **Ответ:** С помощью ключевых слов `virtual` и `override`.
    
4. Что означает ключевое слово `virtual`?  
    **Ответ:** Оно указывает, что метод может быть переопределён в производном классе.
    
5. Чем отличается `override` от `new`?  
    **Ответ:** `override` переопределяет метод, а `new` скрывает метод базового класса.
    
6. Можно ли переопределить статический метод?  
    **Ответ:** Нет, статические методы не могут быть виртуальными.
    
7. Что такое перегрузка методов?  
    **Ответ:** Определение нескольких методов с одним именем, но разными параметрами.
    
8. Как работает ключевое слово `abstract` в полиморфизме?  
    **Ответ:** Оно заставляет производные классы реализовать абстрактный метод.
    
9. Можно ли использовать полиморфизм с интерфейсами?  
    **Ответ:** Да, через реализацию интерфейсов в разных классах.
    
10. Что такое позднее связывание?  
    **Ответ:** Процесс вызова метода определяется в рантайме.
    
11. Какие преимущества даёт полиморфизм?  
    **Ответ:** Универсальность и уменьшение зависимости между классами.
    
12. Как полиморфизм упрощает масштабирование системы?  
    **Ответ:** Позволяет добавлять новые классы без изменения существующего кода.
    
13. Что произойдёт, если не переопределить виртуальный метод?  
    **Ответ:** Будет использована версия метода из базового класса.
    
14. Чем отличается интерфейс от абстрактного класса в контексте полиморфизма?  
    **Ответ:** Интерфейс задаёт контракт, а абстрактный класс может предоставлять частичную реализацию.
    
15. Можно ли использовать ключевое слово `virtual` с конструкторами?  
    **Ответ:** Нет, конструкторы не могут быть виртуальными.
    
16. Что такое "слабая" и "сильная" связанность?  
    **Ответ:** Слабая связанность — классы меньше зависят друг от друга, что достигается через полиморфизм.
    
17. Можно ли вызвать метод базового класса, если он переопределён?  
    **Ответ:** Да, с помощью ключевого слова `base`.
    
18. Что такое upcasting и downcasting?  
    **Ответ:**
    
    - Upcasting: преобразование производного класса в базовый.
    - Downcasting: преобразование базового класса в производный.
19. Какие ограничения есть при использовании полиморфизма?  
    **Ответ:** Полиморфизм не работает со статическими и закрытыми методами.
    
20. Какой принцип SOLID соответствует полиморфизму?  
    **Ответ:** Принцип замещения Лисков (Liskov Substitution Principle).
