Да, это одна из самых обсуждаемых "особенностей" JavaScript, и действительно может показаться запутанной. Давайте разберёмся, почему так происходит, откуда берётся это поведение, и как с ним работать.

---

### **Почему `typeof null === 'object'`?**

История этой особенности берёт начало из ранних версий JavaScript. Когда JavaScript только создавался, значение `null` было задумано как "указатель на отсутствующий объект". В ранней реализации значения в JavaScript представлялись как бинарные структуры с заголовком, в котором хранилась информация о типе данных.

- **Объекты** идентифицировались специальным битовым паттерном, начинающимся с 0.  
- **`null`** случайно попал под этот паттерн, хотя фактически не является объектом.

Из-за этой технической ошибки `typeof null` возвращает `'object'`. Исправление этого поведения сломало бы совместимость с огромным количеством существующего кода, поэтому оно осталось в языке как часть спецификации.

---

### **Почему `Object.__proto__ === null`?**

Здесь мы обращаемся к прототипной цепочке JavaScript. В этой цепочке **`null` — это конечная точка**, верхушка цепочки прототипов.  
- `Object.prototype` — это объект, с которого начинаются все обычные объекты.  
- У `Object.prototype` нет родителя, поэтому его `__proto__` равно `null`.  

Это концептуально верно: **`null` обозначает "отсутствие объекта" или "ничего".**

---

### **Противоречие: null как "ничего" и typeof 'object'**

Проблема заключается в том, что:
- С одной стороны, `null` обозначает "ничего", а не объект.
- С другой стороны, результат `typeof null` утверждает, что `null` — это "object".

Это действительно вызывает путаницу, но понимание контекста помогает. Нужно помнить, что **`typeof` — это оператор, который может давать странные результаты из-за исторических особенностей**.

---

### **Как с этим работать?**

1. **Явная проверка на `null`**  
   Чтобы проверить значение `null`, лучше использовать строгую проверку:
   ```javascript
   const value = null;
   console.log(value === null); // true
   ```

2. **Избегайте полагаться на `typeof` для проверки `null`.**
   ```javascript
   console.log(typeof null === 'object'); // true (может быть сбивающим с толку)
   ```

3. **Проверка на объект**
   Если вы хотите убедиться, что значение является настоящим объектом, можно использовать:
   ```javascript
   const isObject = value => value !== null && typeof value === 'object';
   console.log(isObject(null)); // false
   ```

---

### **Итак, где логика?**
- **`null` — конечная точка цепочки прототипов.**
  - Это делает `null` фундаментальной частью механизма прототипов.
  - Логично, что `Object.__proto__` равно `null`, чтобы завершить цепочку.

- **`typeof null === 'object'` — ошибка реализации.**
  - Это историческое наследие, которое исправить нельзя, чтобы не нарушить старый код.

---

### **Вывод**
`null` действительно играет двойную роль в JavaScript:
1. Как "ничего" (отсутствие значения).
2. Как завершение цепочки прототипов.

Хотя результат `typeof null` сбивает с толку, понимание истории и специфики JavaScript позволяет избежать путаницы. Вместо полагания на `typeof`, используйте явные проверки для большего контроля.
